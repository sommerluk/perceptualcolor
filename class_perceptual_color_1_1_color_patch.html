Detailed Description

A color display widget.

This widget simply displays a color. Useful for showing a selected color. The instantiation and usage is simple. Example:

PerceptualColor::ColorPatch *m_patch = new PerceptualColor::ColorPatch;
m_patch->setColor(Qt::blue);
Note
A similar functionality is available as KColorPatch, but this is part of KDELibs4Support which is available in KF5 but should not be used for new code, but only for legacy code. Also, depending on KDELibs4Support would pull-in another dependency which has been avoided for Tier1 libraries. Furthermore, KColorPatch has support for drag-and-drop, which is not desirable for our use case. Therefore, with ColorPatch there is a lightwise alternative (that, by the way, implements the full API of KColorPatch).

Property Documentation

◆ color

QColor PerceptualColor::ColorPatch::color
readwrite

The color that is displayed.

Default value is an invalid color.

  • If the color is invalid, nothing is displayed. The frame is empty: Only the default widget background is visible within the frame.
  • If the color is valid, the widget frame is filled with this color. If this color is not fully opaque, the background behind the color will be a special background pattern (and not the default widget background).
