Detailed Description

A widget for selecting chroma and hue in LCh color space.

This widget displays the plan of chroma and hue (that means a diagram of the radial and the angle of the LCh color space respectively the a axis and the b axis of the Lab color model) at a given lightness.

The widget allows to select a color within the specified gamut at the given lightness. It reacts on mouse events and on keyboard events (see keyPressEvent() for details).

The circle form of the widget and the selection marker that always indicates the distance from the center of the diagram, all this helps the user to understand intuitively that he is moving within a polar coordinate system and to capture easily the current radial and angle.

Note
This widget always accepts focus by a mouse click within the circle. This happens regardless of the QWidget::focusPolicy property. If you set the QWidget::focusPolicy property to a value that accepts focus by mouse click, the focus will not only be accepted for clicks within the actuel circle, but also for clicks within the surrounding rectangle.

◆ mouseMoveEvent()

void PerceptualColor::ChromaHueDiagram::mouseMoveEvent ( QMouseEvent *  event)
overrideprotectedvirtual

React on a mouse move event.

Reimplemented from base class.

Reacts only on mouse move events m_isMouseEventActive is true.

  • If the mouse moves within the gamut, the diagram’s marker is displaced there. The mouse cursor is invisible; only the diagram’ marker is visible.
  • If the mouse moves outside the gamut, the diagram’s marker always stays within the gamut: The chroma value is sacrified while the hue value is retained. Both, the diagram’s marker and the mouse cursor are visible.

If m_isMouseEventActive is false, it simply falls back to QWidget’s default implementation.

◆ mouseReleaseEvent()

void PerceptualColor::ChromaHueDiagram::mouseReleaseEvent ( QMouseEvent *  event)
overrideprotectedvirtual

React on a mouse release event.

Reimplemented from base class. Reacts on all clicks (left, middle, right).

If m_isMouseEventActive is true then:

  • If the mouse is within the gamut, the diagram’s marker is displaced there.
  • If the mouse moves outside the gamut, the diagram’s marker always stays within the gamut: The chroma value is sacrified while the hue value is retained.
  • The mouse curser is made visible (if he wasn’t yet visible anyway).
  • m_isMouseEventActive is set to false.

If m_isMouseEventActive is false, it simply falls back to QWidget’s default implementation.

◆ setColorFromImageCoordinates()

void PerceptualColor::ChromaHueDiagram::setColorFromImageCoordinates ( const QPoint  imageCoordinates)
private

Sets the color property corresponding to given widget coordinates.

Parameters
imageCoordinatesA coordinte pair within the widget’s coordinate system. This does not necessarily need to be within the actual displayed diagram or even the gamut itself. It might even be negative.
Postcondition
If the widget coordinates are within the represented gamut, then the color property is set correcpondingly. If the coordinates are outside the gamut, then the chroma value is reduced (while the hue is maintained) until arriving at the outer shell of the gamut; this adapted color is than used for the color property.
Note
This function works independently of the actually display color gamut diagram. So if parts of the gamut are cut off in the diagram, this does not influence this function.

◆ updateDiagramCache()

void PerceptualColor::ChromaHueDiagram::updateDiagramCache ( )
private

Refresh the diagram and associated data.

This class has a cache of various data related to the diagram

  • m_diagramImage

This data is cached because it is often needed and it would be expensive to calculate it again and again on the fly.

Calling this function updates this cached data. This is always necessary if the data the diagram relies on, has changed. Call this function always before using the cached data!

This function does not repaint the widget! After calling this function, you have to call manually update() to schedule a re-paint of the widget, if you wish so.

◆ m_isMouseEventActive

bool PerceptualColor::ChromaHueDiagram::m_isMouseEventActive = false
private

Holds if currently a mouse event is active or not.

Default value is false.

  • A mouse event gets typically actived on a mousePressEvent() done within the gamut diagram. The value is set to true.
  • While active, all mouseMoveEvent() will move the diagram’s color marker.
  • Once a mouseReleaseEvent() occurs, the value is set to false. Further mouse movements will not move the marker anymore.
