Detailed Description

template<typename T>
class PerceptualColor::ConstPropagatingRawPointer< T >

A const propagating raw pointer.

With normal raw C++ pointers (and also with std::unique_ptr pointers), within const functions you can do non-const operations on objects that a pointer points to.

This pointer type is different: It propagates the const-ness of the object members and propagates them to the call through the pointer; it will trigger a compiler error if non-cost access to object members or methods is done from within const functions. Apart from that, it behaves similar to raw pointers. For compatibility with raw pointers, it also casts implicitly to the corresponding raw pointer (but only within non-const contexts).

Think of this template as a simple alternative to std::experimental::propagate_const< T* >

Example code:

// Assuming you have a member variable in your class:
PerceptualColor::ConstPropagatingRawPointer<QRectF> pointerToQRectF {new QRectF};
// Now, you access this member variable from a method within your class:
// Helper variables
QRectF myRectF;
qreal myHeight;
QRectF *normalCppPointerToQRectF;
// The following code works within both, const and non-const contexts:
myHeight = pointerToQRectF->height();
myRectF = *pointerToQRectF;
// The following code works only within const contexts.
// Within non-cost contexts, you will get an error at compile time.
pointerToQRectF->setHeight(5);
*pointerToQRectF = myRectF;
normalCppPointerToQRectF = pointerToQRectF;
// Deleting works like for normal pointers
delete pointerToQRectF;
Note
A ConstPropagatingRawPointer pointer variable itself may not be const! (Otherwise, this would make const all access even to non-const functions of the pointed object.)
See also
ConstPropagatingUniquePointer
Note
There exist very sophisticated implementations like https://github.com/jbcoe/propagate_const but we use nevertheless our own lightwise implementation because this seems to be enough for our limited use case.
