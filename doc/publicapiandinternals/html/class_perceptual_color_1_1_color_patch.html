Detailed Description

A color display widget.

This widget simply displays a color. Useful for showing a given color. The instantiation and usage is simple. Example:

PerceptualColor::ColorPatch *m_patch = new PerceptualColor::ColorPatch;
m_patch->setColor(Qt::blue);
ColorPatch

This widget is also able to display transparency:

ColorPatch with 50% transparency

There is also a representation for QColor::isValid() == true:

ColorPatch with invalid color

The default minimum size of this widget is similar to a QToolButton. Depending on your use case, you might set a bigger minimum size:

m_patch->setMinimumSize(QSize(50, 50));

This class’s API is similar to KColorPatch’s API.

Note
A similar functionality is available as KColorPatch, but this is part of KDELibs4Support which is available in KF5 but should not be used for new code, but only for legacy code. Also, depending on KDELibs4Support would pull-in another dependency. Furthermore, KColorPatch has support for drag-and-drop, which is not desirable for our use case. Therefore, with ColorPatch there is a lightweight alternative (that, by the way, implements the full API of KColorPatch).

Property Documentation

◆ color

QColor PerceptualColor::ColorPatch::color
readwrite

The color that is displayed.

Default value is an invalid color.

  • If the color is invalid, nothing is displayed. The frame is empty: Only the default widget background is visible within the frame.
  • If the color is valid, the widget frame is filled with this color. If this color is not fully opaque, the background behind the color will be a special background pattern (and not the default widget background).
Note
No color management is applied. The color is used as-is to paint on the canvas provided by the operation system.
