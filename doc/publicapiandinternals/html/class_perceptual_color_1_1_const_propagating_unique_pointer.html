Detailed Description

template<typename T>
class PerceptualColor::ConstPropagatingUniquePointer< T >

A const propagating std::unique_ptr

With normal std::unique_ptr pointers (and also with raw C++ pointers), within const functions you can do non-const operations on objects that a pointer points to.

This pointer type is different: It propagates the const-ness of the object members and propagates them to the call through the pointer; it will trigger a compiler error if non-cost access to object members or methods is done from within const functions. Apart from that, it behaves like std::unique_ptr (from which it inherits).

Think of this template as a simple alternative to std::experimental::propagate_const< std::unique_ptr<T> >

Example code:

// A ConstPropagatingUniquePointer pointing to a new QObject
PerceptualColor::ConstPropagatingUniquePointer<QObject> myPointer(new QObject());

Currently, move-assignment is not supported. Use std::unique_prt::reset() instead. As there is no support for deleters anyway, this should be equivalent.

Note
A ConstPropagatingUniquePointer pointer variable itself may not be const!

(Otherwise, this would make const all access even to non-const functions of the pointed object.)

See also
ConstPropagatingRawPointer
This code is based on the idea in deep_const_ptr which is a more general implementation of this concept, that does not provide a std::unique_ptr but acts as a wrapper template around smart pointers in general. It lacks however the dereference operator.
