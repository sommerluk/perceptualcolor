◆ PERCEPTUALCOLOR_IMPORTEXPORT

#define PERCEPTUALCOLOR_IMPORTEXPORT   Q_DECL_IMPORT

A macro that either exports or imports library symbols.

This approach is proposed in Qt’s documentation (chapter “Creating Shared Libraries”) – see there for more details. In short:

When your application is build against the dynamic library and includes the header files of the library, the macro imports the corresponding symbols of the library for you by expanding to Qt’s Q_DECL_IMPORT macro. This is the default behaviour of this macro (happens always when PERCEPTUALCOLORLIB_BUILD_DYNAMIC_LIBRARY is not defined).

When the dynamic library itself is build, the macro exports the corresponding symbol by expanding to Qt’s Q_DECL_EXPORT macro. Exported symbols will be visible symbols in the dynamic library. To get this behaviour, it is necessary to define PERCEPTUALCOLORLIB_BUILD_DYNAMIC_LIBRARY always when this library itself is build. The definition can be made within CMake:

target_compile_definitions(PERCEPTUALCOLORLIB_BUILD_DYNAMIC_LIBRARY)

And you also have to make sure that all symbols that are not explicitly exported will be actually hidden:

set_target_properties(perceptualcolor PROPERTIES CXX_VISIBILITY_PRESET hidden)
