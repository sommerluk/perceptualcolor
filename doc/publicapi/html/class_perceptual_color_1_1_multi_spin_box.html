Detailed Description

A spin box that can hold multiple sections (each with its own value) at the same time.

This widget is similar to QDateTimeEdit which also provides multiple sections (day, month, year…) within a single spin box. However, this widget is flexible. You can define on your own the behaviour of each section.

MultiSpinBox

This widget works with floating point precision. You can set the number of decimal places for each section individually, via MultiSpinBoxSectionConfiguration::decimals. (This value can also be 0 to get integer-like behavoir.)

Example code to create a MultiSpinBox for a HSV color value (Hue 0°–360°, Saturation 0–255, Value 0–255) comes here:

PerceptualColor::MultiSpinBox *myHsvSpinBox = new PerceptualColor::MultiSpinBox();
PerceptualColor::MultiSpinBoxSectionConfiguration myConfiguration;
QList<PerceptualColor::MultiSpinBoxSectionConfiguration> hsvConfigurations;
myConfiguration.setDecimals(1);
myConfiguration.setPrefix(QString());
myConfiguration.setMinimum(0);
myConfiguration.setWrapping(true);
myConfiguration.setMaximum(360);
myConfiguration.setSuffix(QStringLiteral(u"° "));
hsvConfigurations.append(myConfiguration);
myConfiguration.setPrefix(QStringLiteral(u" "));
myConfiguration.setMinimum(0);
myConfiguration.setMaximum(255);
myConfiguration.setWrapping(false);
myConfiguration.setSuffix(QStringLiteral(u" "));
hsvConfigurations.append(myConfiguration);
myConfiguration.setSuffix(QString());
hsvConfigurations.append(myConfiguration);
myHsvSpinBox->setSectionConfigurations(hsvConfigurations);
myHsvSpinBox->setSectionValues(QList<double> {310, 200, 100});
// Initial content is: 310,0° 200,0 100,0

You can also have additional buttons within the spin box via the addActionButton() function.

Note
This class inherits from QAbstractSpinBox, but some parts of the parent class’s API are not supported in this class. Do not use them:
  • selectAll() does not work as expected.
  • wrapping() is ignored. Instead, you can configures the wrapping individually for each section via MultiSpinBoxSectionConfiguration::isWrapping.
  • specialValue() is not supported.
  • hasAcceptableInput() is not guaranteed to obey to a particular and stable semantic.
  • fixup(), interpretText(), validate() are not used nor do they anything.
  • keyboardTracking() is ignored. See the signal sectionValuesChanged for details.
  • correctionMode() is ignored.
  • isGroupSeparatorShown is ignored.

Focus handling for Tab respectively Shift+Tab.

Reimplemented from base class.

Note
If it’s about moving the focus within this widget, the focus move is actually done. If it’s about moving the focus to another widget, the focus move is not actually done. The documentation of the base class is not very detailed. This reimplementation does not exactly behave as the documentation of the base class suggests. Especially, it handles directly the focus move within the widget itself. This was, however, the only working solution we found.
Parameters

Returns
true if the focus has actually been moved within this widget or if a move to another widget is possible. false otherwise.

Notify signal for property sectionValues.

This signal is emitted whenever the value in one or more sections changes.

Parameters
newSectionValuesthe new sectionValues

Depending on your use case (for example if you want to use for queued signal-slot connections), you might consider calling qRegisterMetaType() for this type, once you have a QApplication object.

Note
The property keyboardTracking() of the base class is currently ignored. Keyboard tracking is always enabled: The spinbox emits this signal while the new value is being entered from the keyboard – one signal for each key stroke.
